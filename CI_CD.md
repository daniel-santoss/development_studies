# CI / CD (Prática no desenvolvimento de software para automatizar e agilizar o processo de desenvolvimento)

# Continuous Integration (Integração Contínua) - CI
- Integração frequente do código produzido por membros de uma equipe em um repositório compartilhado
- Visa garantir o desenvolvimento de cada membro do time não quebre a aplicação;
- Toda vez que o código é integrada, tarefas são realizadas para verificar a integridade do novo código e identificar problemas antes do código chegar a produção.

# Benefícios do CI:
- Rápida detecção de problemas;
- Agilidade no desenvolvimento;
- Melhoria na qualidade do código;
- Redução de conflitos de código;
- Feedback rápido e contínuo;

# Continuous Delivery (Entrega Contínua) - CD
- Complemento do processo de CI;
- Após garantir que código é integro, o processo de implantação do software ambientes de teste, homologação, etc. é automatizado;
- Garante que aplicação esteja pronta para entrega ao usuário final;
- Reduz o tempo entre o desenvolvimento e a entrega no ambiente

# Continuous Deployment (Implementação Contínua) - CD
- Passo além da Entrega Contínua
- Tem como objetivo também automatizar a implantação em produção;
- Após passar por todas etapas anteriores, automaticamente será implantado no ambiente de produção sem intervenção manual.
	
# Benefícios do CD:
- Implantações rápidas e confiáveis;
- Erros humanos minimizados;
- Melhoria na qualidade do código;
- Detecção imediata de problemas;
- Foco na entrega de valor, devido as automações.
